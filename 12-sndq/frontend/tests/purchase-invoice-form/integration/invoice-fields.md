# Invoice Fields

**Status**: Not started
**Priority**: MEDIUM (wrong invoice number format or dates outside legal bounds)
**Test tier**: Component integration
**Target file**: `src/modules/financial/forms/purchase-invoice-v3/__tests__/integration/invoice-fields.test.tsx`
**Component(s) under test**: `InvoiceFieldsSection` from `../sections/InvoiceFieldsSection.tsx`

## Purpose

Guard the **rendered interaction** of the invoice number auto-generate toggle, date picker, and AI confidence indicators. Pure logic (prefixes, date bounds, `generateInvoiceNumber`) is covered by `constants.test.ts`; these tests verify that clicking buttons and typing in fields correctly wires to `setValue`, `markFieldReviewed`, and the auto-number hook state.

## Risk

Auto-generate button doesn't populate the field, manual edit doesn't clear auto-generated state, AI indicators remain visible after user reviews a field, date picker doesn't format correctly for the API.

## Bugs Guarded

- "Auto-generate populates field" guards auto-number toggle — verifies the rendered toggle button calls `useAutoGeneratedInvoiceNumber.toggle()` which sets the invoice number on the form via `setValue`
- "Toggle off clears field" guards auto-number toggle — second toggle must clear value to empty string
- "Manual typing resets auto state" guards auto-number toggle — the `onChange` handler on the input must call `markFieldReviewed` + `autoNumber.reset()`
- "AI indicator visible/hidden" guards confidence lifecycle — conditional rendering of `AiFieldIndicator` based on `getFieldConfidence` return value

## Scenarios

| Test Name | Expected Outcome |
|-----------|------------------|
| Renders invoice number input and auto-generate button | Input and "Auto" button visible |
| Click auto-generate populates invoice number field | Input value matches `PREFIX-YEAR-NNN` pattern |
| Click auto-generate again clears the field | Input value becomes empty |
| Auto-generate button label toggles | Shows "Auto" when off, "Clear" when on |
| Type in invoice number resets auto-generated state | Subsequent toggle generates a new number (not stuck in "clear" mode) |
| AI confidence indicator visible when confidence defined | `AiFieldIndicator` rendered next to label |
| AI confidence indicator hidden when confidence undefined | No `AiFieldIndicator` in DOM |
| Date picker renders with bounds from props | `fromDate` and `toDate` attributes set |

## Click auto-generate populates invoice number field

**Preconditions**: Invoice number field is empty, auto-generate is off.

### Steps

1. Render `InvoiceFieldsSection` with `mode="invoice"`
2. Click the "Auto" button

### Expected Outcome

- The invoice number input now has a value matching `INV-{YEAR}-{3 digits}`
- The button label changes to "Clear"

### Example Code

```typescript
it('click auto-generate populates invoice number with correct format', async () => {
  const user = userEvent.setup();
  renderWithProviders(
    <InvoiceFieldsSection mode="invoice" />,
  );

  const autoButton = screen.getByRole('button', { name: /auto/i });
  await user.click(autoButton);

  const input = screen.getByRole('textbox', { name: /invoice number/i });
  const year = new Date().getFullYear();
  expect(input).toHaveValue(expect.stringMatching(new RegExp(`^INV-${year}-\\d{3}$`)));
});
```

## Type in invoice number resets auto-generated state

**Preconditions**: Auto-generate is active (number populated).

### Steps

1. Click "Auto" to generate a number
2. Type manually into the invoice number input
3. Observe the button returns to "Auto" state

### Expected Outcome

- `markFieldReviewed('invoiceNumber')` is called on the context
- `autoNumber.reset()` fires, so the button shows "Auto" again (not stuck in "Clear")

### Example Code

```typescript
it('manual typing resets auto-generated state and calls markFieldReviewed', async () => {
  const user = userEvent.setup();
  const markFieldReviewed = vi.fn();
  const { contextValue } = renderWithProviders(
    <InvoiceFieldsSection mode="invoice" />,
    {
      aiExtractionOverrides: { markFieldReviewed },
    },
  );

  // Activate auto-generate
  await user.click(screen.getByRole('button', { name: /auto/i }));
  expect(screen.getByText('Clear')).toBeInTheDocument();

  // Type manually into the field
  const input = screen.getByRole('textbox', { name: /invoice number/i });
  await user.clear(input);
  await user.type(input, 'MANUAL-001');

  // Auto state resets — button shows "Auto" again
  expect(screen.getByText('Auto')).toBeInTheDocument();
  expect(markFieldReviewed).toHaveBeenCalledWith('invoiceNumber');
});
```

  renderWithProviders(
    <InvoiceFieldsSection mode="invoice" />,
    {
