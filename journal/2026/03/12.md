## What I Learned
- Circular dependencies between React Hook Form and Zod (especially `z.infer`) can silently degrade build performance and cause heap memory issues — multiple Zod versions in the same repo amplify the problem
- The RADIO framework (Requirements, Architecture, Data Model, Interface, Optimizations) is a structured approach specifically for front end system design interviews, distinct from back-end system design
- Compound Pattern combined with module-specific component extraction can cleanly separate business logic from UI configuration in large-scale form-heavy applications

## Key Insights
- Build performance issues often have multiple root causes stacked together — fixing one (heap memory) can reveal another (circular deps) which reveals yet another (duplicate library versions)
- Front end system design interviews focus on client-side architecture, application state, and UX — not distributed systems. The evaluation criteria heavily weight tradeoff analysis and product sense.
