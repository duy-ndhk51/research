## What I Learned

- JavaScript's prototype-based inheritance model: objects inherit from objects, not from classes — the prototype chain goes Instance → Constructor.prototype → Object.prototype → null
- Closures are functions bundled with their lexical environment — they capture actual variables (not copies), enabling data privacy, function factories, and persistent state
- First-class functions are JavaScript's superpower: assign to variables, pass as arguments, return from functions — this enables higher-order patterns like callbacks, sorting comparators, and event handlers
- The `==` operator has 4 conversion rules (number/string, boolean/any, null/undefined, same type) while `===` has just one rule: same type AND same value
- MVC (Model-View-Controller) pattern applied to a browser game: Model handles state/logic, View handles DOM updates, Controller processes input and coordinates both

## Key Insights

- The Head First methodology (visuals, conversational tone, exercises, redundancy) aligns with cognitive science principles — making learning stick through multiple modalities
- JavaScript's "everything is an object" philosophy unifies the language: functions have properties, arrays are objects, even primitives auto-box to objects when needed
- The progression from object literals → constructors → prototypes mirrors a real developer's growth: first you learn to make things work, then you learn to make them scale
- Event-driven/asynchronous programming is JavaScript's natural paradigm — the language was designed for dynamic browser environments, not linear execution
