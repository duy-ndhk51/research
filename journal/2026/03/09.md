## What I Learned

- **Double-entry accounting**: Every financial transaction is recorded in at least two accounts — one debit and one credit — ensuring the accounting equation (Assets = Liabilities + Equity) always balances
- JavaScript's prototype-based inheritance model: objects inherit from objects, not from classes — the prototype chain goes Instance → Constructor.prototype → Object.prototype → null
- Closures are functions bundled with their lexical environment — they capture actual variables (not copies), enabling data privacy, function factories, and persistent state
- First-class functions are JavaScript's superpower: assign to variables, pass as arguments, return from functions — this enables higher-order patterns like callbacks, sorting comparators, and event handlers
- The `==` operator has 4 conversion rules (number/string, boolean/any, null/undefined, same type) while `===` has just one rule: same type AND same value
