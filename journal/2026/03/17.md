# 2026-03-17

## What I Worked On

### SNDQ Company
- Built **Account Invoice overview** inside the building dashboard — displays aggregated invoice data and a detailed list of **purchase invoices** inside a floating sheet
- Added **Quick Access middle blocks** to the building dashboard — navigation shortcuts to sub-accounting pages (provision fund, invoices, balances, etc.) for faster access to frequently used sections
- Implemented **overall invoice info view** — users can now see invoice summary data without drilling into individual records
- Built **sidebar navigation between buildings** — users can switch between buildings with a sidebar that displays both **"My Buildings"** tab and **"Workspace Buildings"** tab, eliminating the need to go back to the building list page

## Business Impact
- 🚀 **Account Invoice overview with floating sheet details** — property managers can now see the invoice picture for a building at a glance and drill into purchase invoice details without leaving the dashboard. This eliminates back-and-forth navigation that previously disrupted workflow
- ⚡ **Quick Access blocks** — reduces the number of clicks to reach sub-accounting pages from the building dashboard. For users who check multiple sections daily, this compounds into significant time savings
- 🚀 **Overall invoice info view** — provides instant visibility into invoice status without requiring users to open individual records. Managers can spot issues (overdue, unpaid) faster
- 🚀 **Sidebar building navigation with My Buildings + Workspace tabs** — property managers handling multiple buildings no longer need to return to the building list to switch context. This is a major UX improvement for the core daily workflow — switching between buildings is one of the most frequent actions in the platform
- 📈 **Cross-feature integration** — the assign payment flow now works inside the provision fund detail view, the invoice overview uses floating sheets consistently, and the quick access blocks tie everything together. The building dashboard is becoming a true **command center** for property management

## What I Learned
- Building a dashboard that serves as a "command center" requires thinking about information hierarchy — what does the user need to see first, what should be one click away, and what can be deeper
- Sidebar navigation between buildings is a pattern that scales well — it works for both small portfolios ("My Buildings") and large workspaces ("Workspace Buildings") without UI complexity
- Floating sheets used consistently across the dashboard (assign payment, purchase invoices) create a predictable interaction pattern — users learn once and apply everywhere

## Key Insights
- **The building dashboard is the most valuable real estate in the app** — every improvement here has outsized impact because it's where property managers spend 80%+ of their time
- **Quick Access blocks are underrated UX** — they seem simple, but reducing navigation depth by even one click for a frequently accessed page dramatically improves user satisfaction
- **Sidebar navigation is better than breadcrumb-based navigation** for multi-entity contexts — breadcrumbs force a linear back-and-forth, while a sidebar allows instant lateral movement between buildings
- The combination of floating sheets + sidebar navigation + quick access creates a **hub-and-spoke architecture** for the UI — the building dashboard is the hub, and everything else is reachable without losing context
