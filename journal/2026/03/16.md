# 2026-03-16

## What I Worked On

### SNDQ Company
- Refactored the **Assign Payment flow** using the new **floating sheet** component — replaced the old navigation-based approach with an in-context floating sheet UX
- Updated the **Assign Payment UI** for both **steward** and **syndic** user roles — each role now has a tailored UX path within the same flow
- Started enhancing the **full detailed building dashboard** — organized **skeleton loading patterns** to improve perceived performance during data fetching
- Built **Provision Fund** detailed view inside the building dashboard — displays detailed transactions with filter capabilities and allows users to trigger the assign payment flow directly without navigating away

## Business Impact
- 🚀 **Refactored Assign Payment to floating sheet** — users can now assign payments in-context without losing their place in the dashboard. This reduces friction in one of the most frequent accounting workflows
- 📈 **Role-specific UX for steward and syndic** — each user type sees the flow optimized for their responsibilities, reducing confusion and support tickets from users encountering irrelevant options
- ⚡ **Skeleton loading patterns** — improved perceived load time for the building dashboard. Users see structured placeholders instead of blank screens, which reduces abandonment and builds trust in the platform's responsiveness
- 🚀 **Provision Fund with in-context assign payment** — accountants can view fund transactions, filter them, and assign payments all in one place. Previously this required navigating between multiple pages, costing time on every interaction

## What I Learned
- Floating sheets provide a better UX than full-page navigation for sub-workflows — users maintain context of where they came from, which is critical for accounting flows where you're working within a specific building's data
- Designing role-specific UX (steward vs syndic) requires understanding the business process from each role's perspective — the same data looks different depending on who is looking at it and what decisions they need to make
- Skeleton loading is an investment in perceived performance — even if actual load time doesn't change, users perceive the app as faster when they see structured placeholders instead of spinners or blank screens

## Key Insights
- **In-context workflows beat navigation** — every time you force a user to navigate away and come back, you lose cognitive context. Floating sheets keep the user anchored while enabling complex sub-tasks
- **Role-based UX is a multiplier for multi-tenant platforms** — SNDQ serves different types of property management professionals. Tailoring the experience to each role reduces training time and increases adoption
- Building dashboards are the **daily driver** for property managers — investing in skeleton loading and in-context flows here has outsized impact because this is where users spend most of their time
